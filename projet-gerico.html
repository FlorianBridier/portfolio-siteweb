Gerico — Gestion de congés & fiches de paie

Application web réalisée dans le cadre d’un projet universitaire. L’objectif était de concevoir une plateforme RH permettant de gérer des employés, des demandes de congés et des fiches de paie à partir de données fictives.

Résumé technique

  • Type : application web RH
  • Contexte : projet universitaire SAE
  • Langage : PHP
  • Templates : Smarty
  • Front : HTML, CSS, Bootstrap
  • Données : base SQL
  • Fonctions clés : rôles, congés, employés, paie

Objectif

À quoi sert le projet ?

Gerico est une application web de gestion RH conçue pour centraliser plusieurs besoins d’une organisation : identification des utilisateurs, gestion des employés, suivi des demandes de congés et accès aux fiches de paie.

Le projet repose sur des données fictives, mais reprend une logique proche d’un outil interne d’entreprise : chaque utilisateur dispose d’un rôle et accède aux fonctionnalités adaptées à son profil.

Fonctionnalités

Ce que permet l’application

Authentification

Connexion utilisateur avec distinction des accès selon le rôle : administrateur ou employé.

Gestion des rôles

Interface pensée pour séparer les droits d’administration des actions disponibles côté employé.

Demandes de congés

Création, validation, refus et suivi des demandes avec affichage de l’état de traitement.

Gestion des employés

Consultation des employés, accès aux détails et actions d’administration liées au personnel.

Fiches de paie

Génération ou consultation de fiches de paie à partir de données fictives structurées en base.

Tableau de bord

Vue synthétique permettant d’accéder rapidement aux informations principales de gestion.

Architecture

Organisation technique

L’application suit une logique web classique : une couche PHP pour le traitement, des templates Smarty pour structurer l’affichage, une base SQL pour stocker les données et une interface HTML/CSS/Bootstrap pour proposer une navigation claire.

Utilisateur Interface Bootstrap PHP / Smarty Base SQL

Sécurité & logique métier

Points importants

  • Contrôle des accès selon le rôle de l’utilisateur.
  • Séparation entre l’espace administrateur et l’espace employé.
  • Structuration des données autour des employés, congés et fiches de paie.
  • Interface d’administration pour centraliser les actions importantes.
  • Travail sur la cohérence fonctionnelle d’un outil interne RH.
